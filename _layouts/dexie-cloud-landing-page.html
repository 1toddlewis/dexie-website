Get Started

This guide assumes your existing app use dexie and maintain dependencies via npm. If you're unfamiliar with all that, try read Dexie.js getting started guide and webpack's get started guide.

1. Create your database in the cloud

1. Create your db in the cloud

npx dexie-cloud create

The URL of your database will output to the console.

2. Whitelist your app origin(s)

npx dexie-cloud whitelist http://localhost:8080

3. Install dexie-cloud-addon

npm install dexie-cloud-addon

4. Update your DB declaration

  import Dexie from "dexie";
  import DexieCloud from "dexie-cloud-addon";

  // Declare your DB
  const db = new Dexie('mySyncedDB', {addons: [DexieCloud]});
  db.version(1).stores({
    todoLists: '$$id, title',
    todoItems: '$$id, title, done, todoListId'
  });

  // Connect your dexie-cloud database:
  db.cloud.connect("https://<yourdatabase>.dexie.cloud", {
    requireAuth: true
  });

Use the URL that was printed out in step 1.

5. Done!

Enjoy.

Your db facing code will continue work like before: query and update the db as you normally do with plain Dexie.js.

With this setup, users will be requested to login using passwordless email OTP the very first time they visit your app from a device or browser. If you already have another authentication in place, or want to integrate with the authentication of your choice, this can be defined in the options object passed to db.cloud.connect(). The API docs for Dexie Cloud will provide more details when released...

Your app will now instantly sync changes with the cloud. If offline, changes are queued until next time you are online.

Without taking advantage of the built-in access control tables, all data will be regarded private for the end user and stored on a per-user basis. Improve your app by allowing users to share things to their team or friends.
