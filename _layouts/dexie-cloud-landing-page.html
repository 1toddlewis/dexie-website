Get started

This guide assumes your existing app use dexie and maintain dependencies via npm. If you're unfamiliar with all that, try read Dexie.js getting started guide and webpack's get started guide.

1. Create your database in the cloud

1. Create your db in the cloud

npx dexie-cloud create

The URL of your database will output to the console.

2. Whitelist your app origin(s)

npx dexie-cloud whitelist http://localhost:8080

3. Install dexie-cloud-addon

npm install dexie-cloud-addon

4. Update your DB declaration

  import Dexie from "dexie";
  import dexieCloud from "dexie-cloud-addon";

  // Declare your DB
  const db = new Dexie('mySyncedDB', {addons: [dexieCloud]});
  db.version(1).stores({
    todoLists: 'id, title',
    todoItems: 'id, title, done, todoListId'
  });

  // Connect your dexie-cloud database:
  db.cloud.connect("https://<yourdatabase>.dexie.cloud", {
    requireAuth: true
  });

The URL "https://<yourDatabase>.dexie.cloud" is the one printed out in step 1.

5. Done!

Enjoy.

Use your db with the simple Dexie API you are used to. Just make sure the IDs you generate are globally unique and don't use auto-incremented primary keys. There's a helper function newId() in dexie-cloud-helpers that generates globally unique strings for you. 

npm install dexie-cloud-helpers
import { newId } from "dexie-cloud-helpers";

// Create objects
db.todoLists.add({
  id: newId(),
  title
});

Use the same Dexie.js API reference you are used to for manipulating and querying data.

Changes sync instantly with the cloud in both directions. When offline, changes are queued until next time you are online. When online, changes will be synced eagerly in both directions, allowing also things like online collaboration. If user is idle for a time, the client will go over to period sync strategy instead of the eager sync. As soon as user starts interact with the application (mouse moves or key strokes), the app will go over to eager sync mode again.

With this setup, users will be requested to login using passwordless email OTP the very first time they visit your app from a device or browser. Learn how to customize authentication if already have another authentication in place, or want to integrate with the authentication of your choice.

Without taking advantage of the built-in access control tables, all data will be regarded private for the end user and stored on a per-user basis. Improve your app by allowing users to share things to their team or friends.
